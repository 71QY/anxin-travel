# 安心出行 - 安装使用说明

## 一、环境要求

- **后端**: JDK 17.0.18+, Maven 3.6+
- **数据库**: MySQL 5.7+, Redis 6.0+
- **前端**: Android Studio (最新版), Android SDK
- **网络**: 确保前后端设备在同一局域网内（或配置公网 IP）

## 二、后端安装与启动

### 1. 数据库初始化

创建数据库并导入脚本：
```sql
CREATE DATABASE anxin_travel DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE anxin_travel;
SOURCE database/anxin_travel.sql;
```

### 2. 配置文件修改

编辑 `src/main/resources/application.yml`，重点关注以下配置：

*   **数据库连接**：修改 `spring.datasource.username` 和 `password`。
*   **Redis 连接**：修改 `spring.data.redis.host` 和 `port`。
*   **API Key 配置**：
    *   `anxin.tencent.map.key`: 腾讯地图 WebService API Key。
    *   `tongyi.api.key`: 阿里云 DashScope (通义千问) API Key。

### 3. 启动后端

在终端运行：
```bash
mvn clean spring-boot:run
```
启动成功后，后端将监听 `0.0.0.0:8080`，允许局域网访问。

## 三、前端安装与配置

### 1. 获取 API Key

| 服务 | 获取地址 | 用途 |
| :--- | :--- | :--- |
| **腾讯地图** | [腾讯位置服务平台](https://lbs.qq.com/dev/console/key/manage) | 逆地理编码、搜索 |
| **高德地图** | [高德开放平台](https://console.amap.com/dev/key/app) | 路线规划、距离计算 |
| **阿里云 AI** | [DashScope 控制台](https://dashscope.console.aliyun.com/apiKey) | 智能体对话、图片识别 |

### 2. 修改 local.properties

在 Android 项目根目录的 `local.properties` 文件中添加：

```properties
# 1. 你的开发环境路径
sdk.dir=C:\\Users\\YourName\\AppData\\Local\\Android\\Sdk

# 2. 后端接口地址（根据实际运行环境修改）
# 模拟器访问本机后端：http://10.0.2.2:8080
# 真机访问局域网后端：http://192.168.189.80:8080 (请替换为你电脑的实际 IP)
api.base.url=http://192.168.189.80:8080

# 3. API Keys
amap.web.api.key=你的高德Web服务Key
tencent.map.key=你的腾讯地图Key
tongyi.api.key=你的阿里云DashScope Key
```

### 3. 编译运行

使用 Android Studio 打开项目，点击 `Run` 按钮即可安装到模拟器或真机。

## 四、前后端联调指南

为了确保手机端能成功连接后端，请执行以下检查：

1.  **防火墙设置**：确保后端电脑的 Windows 防火墙已放行 `8080` 端口。
2.  **IP 确认**：在后端电脑运行 `ipconfig`，找到 IPv4 地址（如 `192.168.189.80`）。
3.  **连通性测试**：在手机浏览器输入 `http://192.168.189.80:8080`，若能访问则说明网络通畅。
4.  **WebSocket 地址**：前端代码中的 WebSocket 地址应同步更新为 `ws://192.168.189.80:8080/ws/agent`。

## 五、常见问题排查

*   **Q: 手机连不上后端？**
    *   A: 检查 `application.yml` 中 `server.address` 是否为 `0.0.0.0`；检查手机和电脑是否在同一 WiFi 下。
*   **Q: 报错 "Connection reset"？**
    *   A: 通常是传输的图片 Base64 过大。建议优化图片压缩逻辑，或在 `application.yml` 中调大 `max-request-size`。
*   **Q: 司机位置不更新？**
    *   A: 检查订单状态是否为“进行中”；检查后端日志是否有 WebSocket 推送记录。
